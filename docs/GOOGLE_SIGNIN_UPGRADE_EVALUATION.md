# Google Sign-In Upgrade Evaluation

GoogleSignIn-iOS resolves cleanly from 7.1.0 to 9.1.0, moving AppAuth-iOS from 1.7.6 to 2.0.0 and GTMAppAuth from 4.1.1 to 5.0.0; the existing `GIDSignIn.sharedInstance.signIn(withPresenting:)` call sites in `WelcomeView` and `GuestTestContainerView` still compile against the upgraded SDK and continue to expose `result.user.idToken?.tokenString`. The upstream `GoogleSignInSwift/Sources/GoogleSignInButtonStyling.swift` in 9.1.0 still uses the v7-era styling constants: 2-point corners, blue dark scheme, Roboto-Bold, and a gray shadow. The brand-compliance refactor should therefore build a custom Google button instead of reusing the SDK SwiftUI button; the SDK bump can remain as a dependency update, but button reuse would not satisfy the current pill/neutral/Roboto Medium/no-shadow brand target.
